# 考察报告写作指南

## 报告要求

### 主题选择

选择 1-2 个专业主题进行深入分析，避免流水账式的全景记录。

参考方向：
- 建筑语言融合与场所重构
- 材料表达与建造智慧
- 空间秩序与场所精神
- 遗产保护与活态传承
- 中西建筑文化的对话

### 结构建议

1. **引言**：提出研究问题
2. **案例分析**：选择能形成对比逻辑的案例
3. **专业视角**：运用建筑史和保护理论
4. **批判性思考**：不只是描述，要有反思
5. **结论**：提炼理论观点

### 内容要求

- **字数**：2000字左右
- **图片**：不少于10张，包含现场照片和专业图纸
- **逻辑**：从现象到本质，从具体到抽象
- **深度**：体现建筑历史和文化遗产保护的专业性

## 图片处理

### 图片格式

支持格式：
- JPG / JPEG（推荐用于照片）
- PNG（推荐用于带透明背景的图）
- GIF（动图）
- SVG（矢量图）
- WebP（现代格式）

### 图片优化

建议尺寸：
- 宽度：800-1200 像素
- 文件大小：建议小于 2MB
- 格式：JPG 75-85% 压缩质量

工具推荐：
- [TinyPNG](https://tinypng.com/) - 在线压缩
- [Squoosh](https://squoosh.app/) - Google 压缩工具
- Photoshop 或 GIMP - 本地编辑

### 图片插入

在 Markdown 中使用：

```markdown
![图片说明](相对路径/图片文件名.jpg)
```

示例：

```markdown
![图1：立园泮立楼主立面](./images/liyuan-1.jpg)
```

## 协作流程

### 使用 GitHub 协作

1. **Fork 仓库**
   - 点击仓库右上角的 Fork 按钮
   - 创建自己的副本

2. **创建分支**
   ```bash
   git checkout -b feature/你的报告
   ```

3. **提交更改**
   ```bash
   git add .
   git commit -m "添加第X组考察报告"
   ```

4. **推送分支**
   ```bash
   git push origin feature/你的报告
   ```

5. **创建 Pull Request**
   - 在 GitHub 上发起 PR
   - 邀请队友和老师审核

## 常见问题

### Q: 如何确定报告主题？

A: 从以下角度思考：
- 你最感兴趣的问题是什么？
- 哪些案例之间有对比性？
- 你能从哪个专业角度深入分析？

### Q: 图片太多怎么办？

A: 精选原则：
- 选择最能说明问题的图片
- 每个章节 2-3 张图片为宜
- 图文要紧密配合

### Q: 如何提升专业性？

A: 建议方法：
- 引用专业理论和术语
- 绘制分析图和示意图
- 进行批判性思考
- 参考学术文献

### Q: 报告如何修改？

A: 流程：
1. 创建新分支进行修改
2. 提交 Pull Request
3. 团队成员评论反馈
4. 根据反馈修改
5. 合并到主分支
