🌐 部署到 GitHub Pages ​

第一步：创建 GitHub 仓库 ​

  1. 登录 GitHub
  2. 点击右上角 + → New repository
  3. 仓库名：kaiping-diaolou-reports
  4. 选择 Public
  5. 点击 Create repository

第二步：上传代码 ​

bash
cd C:\Users\stone\CodeBuddy\20260331113721
git init
git add .
git commit -m "初始化开平碉楼考察报告网站"
git branch -M main
git remote add origin https://github.com/你的用户名/kaiping-diaolou-reports.git
git push -u origin main

第三步：配置 GitHub Pages ​

  1. 进入仓库的 Settings → Pages
  2. 在 Build and deployment 下：
    • Source：选择 GitHub Actions
  3. 点击 Save

第四步：创建 GitHub Actions 配置 ​

创建文件 .github/workflows/deploy.yml：

yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: write
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install dependencies
        run: npm ci

      - name: Build VitePress site
        run: npm run docs:build

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: .vitepress/dist

      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4

第五步：推送配置 ​

bash
git add .
git commit -m "添加 GitHub Actions 部署配置"
git push

第六步：访问网站 ​

几分钟后，你的网站会部署到： https://你的用户名.github.io/kaiping-diaolou-reports/

👥 团队协作流程 ​

对于学生 ​

第1步：Fork 仓库 ​

  1. 访问你的 GitHub 仓库
  2. 点击右上角 Fork 按钮
  3. 创建自己的仓库副本

bash
git clone https://github.com/你的用户名/kaiping-diaolou-reports.git
cd kaiping-diaolou-reports

第3步：创建分支 ​

bash
git checkout -b feature/第X组报告

  1. 在 reports/ 目录下创建或编辑文件
  2. 将图片放到 public/images/ 目录

git add .

  1. 访问你的 Fork 仓库
  2. 点击 Pull requests → New pull request
  3. 填写标题和描述

第8步：修改并合并 ​

根据反馈修改后，更新 PR，最终合并到主分支

🎨 自定义主题 ​

修改网站标题和描述 ​

编辑 .vitepress/config.ts：

typescript
export default defineConfig({
  title: '你的网站标题',
  description: '你的网站描述',
  // ...
})

修改导航栏 ​

typescript
themeConfig: {
  nav: [
    { text: '首页', link: '/' },
    { text: '指南', link: '/guide/writing-guide' },
    { text: '报告', link: '/reports/' }
  ]
}

添加 Logo ​

在 public/ 目录下放置 logo 图片，然后在配置中引用：

typescript
themeConfig: {
  logo: '/logo.png'
}

📚 VitePress 高级功能 ​

搜索功能 ​

VitePress 内置了搜索功能，默认已启用。

评论功能 ​

VitePress 本身不支持评论，可以集成第三方服务：

  • Giscus - 基于 GitHub Discussions
  • Utterances - 基于 GitHub Issues

自定义样式 ​

创建 .vitepress/style.css：

css
:root {
  --vp-c-brand: #646cff;
  --vp-c-brand-light: #747bff;
}

然后在配置中引入：

typescript
export default defineConfig({
  themeConfig: {
    // ...
  },
  vite: {
    css: {
      postcss: './.vitepress/postcss.config.js'
    }
  }
})

❓ 常见问题 ​

Q: 图片不显示？ ​

A: 检查以下几点：

  • 图片是否放在 public/ 目录下
  • 图片路径是否正确（应该以 / 开头）
  • 图片文件名是否正确（区分大小写）

Q: 如何修改网站颜色？ ​

A: 在 .vitepress/style.css 中修改 CSS 变量

Q: 如何添加多语言支持？ ​

A: 在 .vitepress/config.ts 中配置 locales

Q: 如何优化网站性能？ ​

A:

  • 压缩图片
  • 使用 VitePress 的图片优化功能
  • 减少 JavaScript 依赖
